## Usage

These documents serve as business domain reference for OpenDebt development:
- **Stamdata table** (doc 01): defines claim entity fields and validation rules
- **Reason codes** (docs 04a-04c): define claim lifecycle state transitions
- **Notification types** (doc 06): define integration message types
- **Interest rules** (doc 07): define calculation engine requirements
- **Portal guides**: UX reference for Fordringshaverportal implementation
- **Fordringstype bilag**: claim type validation rules and filter specifications
