# Afstemning (Reconciliation)

Gældsstyrelsen regularly calls for reconciliation of the claim mass in PSRM. Creditors must verify that their records match what is registered in PSRM and report the result.

## Steps

1. **Ensure the "Afstem fordringer" role** is assigned in TastSelv Erhverv.
2. **Download the justeringsrapport** (adjustment report) from the Fordringshaverportalen.
3. **Calculate the sum of corrections** — tilgang (additions), opskrivning (upward adjustments), nedskrivning (downward adjustments), and tilbagekald (recalls).
4. **Submit the afstemningsresultat** (reconciliation result) in the portal under the "Afstemning" tab.

## Afstemningsmetode (Reconciliation Formula)

```
Tilgang − Tilbagekald − Nedskrivning + Opskrivning = Inddrivelsesgrundlag
```

Only the creditor's own transactions are included. Do **not** account for dækninger (recoveries) from Gældsstyrelsen.

## Common Causes of Differences

| Cause | Description |
|-------|-------------|
| Transaction only in PSRM | A transaction exists in PSRM but not in the creditor's books |
| Different values | The same transaction has different amounts in PSRM vs. creditor's records |
| Only in creditor's books | A transaction exists in the creditor's system but not in PSRM |
| Timing differences | Transactions submitted near month-end may appear in different periods |

## Saldoopløsning (Balance Decomposition)

When a difference is identified, break it down to the transaction level:

1. Identify which specific transactions cause the difference.
2. Determine whether the error is in PSRM or in the creditor's own records.
3. Correct errors in PSRM as needed (e.g., nedskrivning, opskrivning, or tilbagekald).
