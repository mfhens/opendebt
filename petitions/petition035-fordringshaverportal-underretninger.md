# Petition 035: Fordringshaverportal -- Underretninger

## Summary

The Fordringshaverportal shall allow creditors to search for, view counts of, and download notifications (underretninger) generated by the debt recovery system, in PDF or XML format.

## Context and motivation

Gaeldsstyrelsen generates notifications when significant events occur on claims -- status changes, payments, decisions, interest calculations. Creditors need access to these for their records and operational processes. The legacy portal provided search and bulk download via a custom notification service. In OpenDebt, notifications are managed by `debt-service` (or a future notification service) and stored in OpenDebt's own storage.

**Supersedes:** Fordringshaverportal petition 009 (old).
**References:** Petition 004 (underretning/paakrav/rykker), Petition 008 (notification preferences in creditor data model), Petition 012 (BFF).

## Functional requirements

### Notification search

1. The portal shall provide a notification search page where creditors can check for available notifications.
2. Search parameters shall include:
   - Date range (from, to)
   - Notification types (checkboxes for each type the creditor is configured to receive)
3. The search result shall display the count of matching notifications.

### Notification download

4. Creditors shall be able to download notifications matching their search criteria.
5. Download format options shall include PDF and XML (selectable via checkboxes).
6. The download shall produce a zip file containing the matching notifications.
7. Download progress shall be indicated to the user (HTMX polling or progress indicator).

### Notification types

8. The available notification types shall be determined by the creditor's notification preferences in the creditor agreement (petition 008):
   - Renteunderretninger (interest notifications)
   - Detaljerede renteunderretninger (detailed interest notifications)
   - Udligningsunderretninger (equalisation notifications)
   - Allokeringsunderretninger (allocation notifications)
   - Afregningsunderretninger (settlement notifications)
   - Returunderretninger (return notifications)
   - Afskrivningsunderretninger (write-off notifications)

### Access control

9. Notifications shall be accessible to all users with `CREDITOR_VIEWER` or `CREDITOR_EDITOR` roles.

## Layout and accessibility

10. The notifications page shall use the SKAT standardlayout.
11. Breadcrumb: Forside > Underretninger.
12. The date range picker shall use standard HTML date inputs with proper labels.
13. Notification type checkboxes shall be grouped in a `<fieldset>` with a `<legend>`.
14. Download status shall be communicated to screen readers via `aria-live` region.
15. All text shall use message bundles (petition 021).

## Data source mapping

| Old source | OpenDebt source |
|---|---|
| NotificationService.search() | `debt-service` GET /api/v1/notifications?creditorId={id}&dateFrom=...&dateTo=... |
| NotificationService.download() | `debt-service` GET /api/v1/notifications/download?... (returns zip) |

## Constraints and assumptions

- Notification storage and generation is a backend concern outside this petition's scope.
- The BFF handles creditor resolution and injects the acting creditor ID.
- Download may be asynchronous with HTMX polling for completion.

## Out of scope

- Notification generation logic
- Backend notification storage design
- Push notifications or real-time notification delivery
